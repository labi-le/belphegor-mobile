# Repository Guidelines

## Project Overview

`belphegor-mobile` is an Apache-2.0 **Android clipboard-sync client** for the [`belphegor`](https://github.com/labi-le/belphegor) P2P clipboard tool. The phone is a first-class peer: it runs the **same Go core** as the desktop, compiled to an Android `.aar` via `gomobile`. No cloud, no account — text and images sync over a LAN mesh (QUIC/TLS 1.3, gated by a shared secret). Min Android 8.0 (API 26).

Three build units:
- **Go core** — upstream, vendored as a git submodule at `./belphegor` (do not edit).
- **AAR bridge** — `gomobile` facade in `./glue`, overlaid onto the core, bound into `app/libs/belphegor.aar`.
- **Kotlin app** — `./app` (`belphegor.app`), the client UI + foreground service.
- **LSPosed module** — `./background-clipboard` (`belphegor.background`), unblocks background clipboard reads on Android 10+.

## Architecture & Data Flow

**gomobile boundary (the API Kotlin actually calls).** `glue/mobile/belphegor.go` (Go `package mobile`) → `gomobile bind` → Java package **`belphegor.mobile`**: static class **`Mobile`** (`newConfig()`, `start(cfg, handler, logs)`) plus types **`Config`**, **`Node`**, **`Handler`**, **`LogSink`**. Go PascalCase methods bind to Java camelCase (`Node.PushClipboard` → `node.pushClipboard`); Go `int` marshals to Java `Long`; callbacks are Kotlin SAM lambdas.

**Runtime wiring.**
- `BelphegorService` (foreground service, `dataSync|specialUse`) owns the `Node` for the process lifetime: `Mobile.newConfig()` → set `Config` fields from `Prefs` → `Mobile.start(cfg, bridge.handler, LogSink{ LogStore.add(it) })` → publishes `NodeState.node`.
- `MainActivity` renders a 4-tab programmatic UI (Dashboard / Nodes / Settings / Logs) and **polls `NodeState` every 1500 ms** on a main-thread `Handler` (no service binding).
- `ClipboardBridge` bridges both ways: `OnPrimaryClipChangedListener` → `node.pushClipboard(mime, bytes)` (out); `Handler{ mimeType, data -> writeLocal }` → `setPrimaryClip` / FileProvider image URI (in).
- `BackgroundClipboard` (LSPosed) runs in `system_server`, hooks `ClipboardService.clipboardAccessAllowed`, and force-returns `true` when a String arg equals `"belphegor.app"`.

```
copy on phone -> ClipboardBridge listener -> node.pushClipboard
   -> Go Node -> QUIC/TLS(secret) -> LAN -> desktop peer
desktop peer -> QUIC/TLS -> Go Node -> Handler.HandleClipboard
   -> ClipboardBridge.writeLocal -> Android clipboard   (background writes need the LSPosed bypass)
LAN discovery: multicast (under Wi-Fi MulticastLock) -> node.discoveredJSON() -> Nodes tab
```

**Load-bearing behaviors to respect:**
- The core does **not** auto-redial dropped *outgoing* links; `BelphegorService`'s watchdog re-dials saved peers every 15 s (also on network change / app resume).
- `NodeID` is host-supplied via `Config.nodeID` (Android SELinux denies the MAC lookup, so every phone would otherwise collide on id 1). `Prefs.nodeId` generates a stable random `2..1023` per install.
- `wifiOnly` pause tears the node down (`NodeState.node = null`) while the service/notification stay up; the UI shows "Paused" via `NodeState.pausedForNetwork`.

## Development Commands

Everything runs inside the repo-root **`nix-shell`** (`shell.nix`), which supplies the exact Go/NDK/SDK/JDK/Gradle toolchain.

```bash
git submodule update --init                 # once: fetch the ./belphegor core
nix-shell --run 'scripts/build-aar.sh'      # gomobile bind -> app/libs/belphegor.aar
nix-shell --run 'scripts/apk.sh'            # AAR + :app:assembleDebug (one-shot, staged logs)
```

Inside an already-active `nix-shell` (README/CI path — builds **both** modules via the wrapper):

```bash
scripts/build-aar.sh
./gradlew :app:assembleDebug :background-clipboard:assembleDebug
```

Release variants (CI signs them; see below):

```bash
./gradlew :app:assembleRelease :background-clipboard:assembleRelease   # ARM, R8-minified
./gradlew :app:assembleRelease -Px86only                              # x86_64 (emulator/Waydroid)
```

- `build-aar.sh` auto-invokes `setup-core.sh` — no separate overlay step.
- `apk.sh` uses the nix-provided `gradle` (`--no-daemon`), app module only; README/CI use `./gradlew` (Gradle 8.9). Both are valid.
- **Install is manual sideloading** — there is no adb script in the repo. Debug output: `app/build/outputs/apk/debug/app-debug.apk`. Conventional (not in repo): `adb install -r app/build/outputs/apk/debug/app-debug.apk`.

## Code Conventions & Common Patterns

- **Modern Kotlin, no Compose / ViewModel / coroutines / data-binding.** Plain `View`s + `java.util.concurrent` executors + `android.os.Handler`. Packages: `belphegor.app`, `belphegor.background`.
- **UI is 100% programmatic** via `Context`/`LinearLayout` extension functions in `Views.kt` — match this, never add layout XML. Examples:
  ```kotlin
  fun Context.card(build: LinearLayout.() -> Unit): MaterialCardView          // inset iOS card
  fun LinearLayout.switchRow(title: String, subtitle: String?, checked: Boolean): MaterialSwitch
  fun LinearLayout.fieldRow(label: String, example: String, value: String, numeric: Boolean = false): EditText
  ```
- **UI rules are mandatory:** `DESIGN.md` is the source of truth for every color/size/pattern (Apple-HIG over Material; tokens in `res/values[-night]/colors.xml`, primitives in `Views.kt`). Settings **apply on change — no Save button**; state is never color-alone.
- **Shared state = `object` singletons:** `NodeState` (`@Volatile var node` + `pausedForNetwork`), `LogStore` (500-line ring buffer), `Updater`. Read by UI without binding.
- **Settings:** `Prefs` wraps one `SharedPreferences` file `"belphegor"`; each knob is a typed `var` backed by a `KEY_*` const; reads are live. A running node picks up changes only on its **next start**.
- **Threading:** main `Handler` poll @ `POLL_MS=1500`; `Executors` single-thread `dialer` for blocking `connect()`; scheduled watchdog @ `WATCHDOG_MS=15000`; `ConnectivityManager.registerDefaultNetworkCallback`; raw `Thread{}` for APK/update IO.
- **Error handling:** pervasive `runCatching { ... }.getOrNull()/onFailure`; `try/catch` only around `Mobile.start`. Per-class `Log` TAGs (`"BelphegorService"`, `"ClipboardBridge"`, `"Belphegor"`, `"belphegor-bg"`); core lines mirror to `LogStore.add("[app] …")`.
- **Go `Config` (`glue/mobile/belphegor.go`)** exposes 14 gomobile-safe fields (secret, port, deviceName, transport `quic|tcp`, discover, maxPeers, nodeID, allowCopyFiles, maxFileSizeBytes, maxClipboardFiles, discoverDelaySec, keepAliveSec, …); `NewConfig()` seeds from `node.DefaultOptions()`.
- **Comments explain _why_, not _what_ — code must be self-documenting.** Prefer precise names over narration; never add comments that restate what the code plainly does (`// increment i`, `// start the service`, `// set title`). Keep only what the code can't say itself: KDoc on classes and the non-obvious rationale this repo depends on (e.g. the `NodeID`/SELinux note, the gomobile version-skew warning, "load-bearing" behavior comments). If a comment just echoes the next line, delete it and fix the name instead.

## Runtime/Tooling Preferences

- **Use the repo-root `nix-shell` (`shell.nix`) — it is the canonical toolchain.** Do not confuse it with `belphegor/shell.nix` (upstream, Go+protobuf only).
- Pinned versions (authoritative in `shell.nix` + `.github/workflows/build.yml`): **Go 1.26** (`GOTOOLCHAIN=local`), **gomobile / `golang.org/x/mobile` `v0.0.0-20241213221354-a87c1cf6cf46`** (version skew crashes the generated binding at init), **NDK 26.1.10909125**, **SDK platform 35 / build-tools 35.0.0**, **JDK 17**, **Gradle 8.9** wrapper, min API 26.
- **Submodule/glue overlay:** `setup-core.sh` does `cp -a glue/. belphegor/` (mapping `glue/mobile/*` → `belphegor/mobile/*`, `glue/internal/node/mobile.go` → core) because Go forbids importing another module's `internal/*`. These land as **untracked** files; `.gitmodules` `ignore=dirty` masks them — **do not remove `ignore=dirty`**, and never commit the `belphegor` gitlink bump casually (it is managed by `track-core-release.yml`). Expect `setup-core.sh` to mutate the submodule's `go.mod`/`go.sum` + `go.work.sum` after a build.
- **No Gradle version catalog** — deps are direct literals in the module `build.gradle.kts`: `androidx.core:core-ktx:1.13.1`, `androidx.appcompat:appcompat:1.7.1`, `com.google.android.material:material:1.14.0`, and `de.robv.android.xposed:api:82` (`compileOnly`).
- **Signing lives in CI only** (`release.yml`, `apksigner` + `KEYSTORE_*` secrets); no `signingConfigs` in Gradle. Unsigned builds ship as `*-unsigned.apk`. `versionCode`/`versionName` in `app/build.gradle.kts` are auto patch-bumped by `track-core-release.yml`; `v*` tags trigger `release.yml`.
- The `.aar` and `*.apk` are gitignored — always **regenerate**, never commit.

## Testing & QA

- **This repo has no automated tests and no lint/coverage gates.** No `app/src/test`, no `androidTest`, no `*Test.kt`, no Go tests in `glue/`, no `testImplementation`/`testInstrumentationRunner`, no detekt/ktlint/jacoco/golangci config. `./gradlew testDebugUnitTest` / `connectedAndroidTest` have nothing to run. CI (`build`/`release`) only assembles/signs — no test or lint step.
- The only static check that ever executes is AGP's implicit `lintVitalRelease` during `assembleRelease` (not a configured gate). The Go tests + `.golangci.yml` under `./belphegor` are **upstream** and are not run here.
- **Therefore, verify changes manually.** Build the debug APK and exercise the actual behavior on a device or emulator (an `x86_64` build exists for emulator/Waydroid). For clipboard/service/LSPosed changes, confirm end-to-end against a running desktop `belphegor` peer (copy phone→desktop and desktop→phone, background and foreground). Any new behavior needs at minimum a hands-on smoke test — there is no suite to catch regressions.
